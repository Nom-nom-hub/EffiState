State

In EffiState, state represents the complete data model of your application at any given point in time.

State Structure

EffiState works with plain JavaScript objects as state. The structure is entirely up to you: 

// Simple counter state
const counterState = {
  count: 0
};

// Todo app state
const todoState = {
  todos: [
    { id: 1, text: 'Learn EffiState', completed: false },
    { id: 2, text: 'Build an app', completed: false }
  ],
  filter: 'all',
  loading: false
};

// Complex application state
const appState = {
  user: {
    id: 'u1',
    name: 'John Smith',
    preferences: {
      theme: 'dark',
      notifications: true
    }
  },
  posts: [],
  ui: {
    sidebar: {
      open: true,
      width: 240
    },
    modal: null
  }
};

State Immutability

By default, EffiState treats state as immutable, creating new copies when updates occur: 

const store = createStore({ count: 0 });

// This creates a new state object
store.set({ count: 1 });

// This will throw an error if immutability is enabled
store.get().count = 1; // Error: Cannot assign to read-only property

Immutability helps prevent bugs and enables performance optimizations through reference equality checks.

Updating State

Use the set method to update state: 

// Replace the entire state
store.set({ count: 1, user: { name: 'Alice' } });

// Merge with existing state (shallow merge)
store.set({ count: store.get().count + 1 });

// Update nested properties
store.set({
  user: {
    ...store.get().user,
    name: 'Bob'
  }
});

For deeply nested updates, you can use the path-based setter: 

// Update a nested property directly
store.setPath('user.preferences.theme', 'light');
